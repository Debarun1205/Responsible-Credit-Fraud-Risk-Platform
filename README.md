

# Responsible Credit & Fraud Risk Platform

An end-to-end system that combines **agentic AI**, **machine learning**, and **data science** into a single deployed application for credit risk scoring, fraud detection, and fairness auditing.

> **Project status:** ✅ Deployed and running. All four modules are built and wired into a single Streamlit app. LLM-powered features (agentic EDA summary, LLM feature extraction, fraud explanations) run in a graceful fallback/demo mode until an `ANTHROPIC_API_KEY` is configured — see [Progress](#progress) and [Running locally](#running-locally).
>
> **Live demo:** `https://responsible-credit-fraud-risk-platform.streamlit.app/`

---

## What this project does

Given a lending or transactions dataset, the platform:

1. **Profiles the data automatically** using a Claude-powered agent that plans and runs its own exploratory analysis.
2. **Scores credit risk** using a model trained on structured features *plus* features extracted from unstructured text (loan purpose, employment title) via the Claude API.
3. **Flags fraudulent transactions** using an anomaly/classification model, with each flag accompanied by a plain-English explanation generated by an LLM.
4. **Audits every model for fairness**, computing subgroup false-positive/false-negative rates across demographic slices and reporting statistically controlled bias findings.

All four pieces are served from one dashboard rather than four separate scripts.

## Where this sits across AI, ML, and DS

| Domain | What lives here |
|---|---|
| **Artificial Intelligence** | The EDA agent (Claude + tool use planning its own analysis), the LLM-based text-to-feature extractor, and the natural-language fraud-flag explainer |
| **Machine Learning** | The credit risk classifier and the fraud/anomaly detection model, including their training, evaluation, and metrics (ROC-AUC, precision/recall) |
| **Data Science** | The exploratory profiling output, the fairness audit (subgroup rate testing, FDR-controlled significance), and the reporting/visualization layer |

This split is explained in more depth in [`docs/domain_mapping.md`](docs/domain_mapping.md).

## Tech stack

| Layer | Choice | Why |
|---|---|---|
| Modeling | scikit-learn (+ optionally XGBoost) | Fast to train, well-understood tradeoffs |
| LLM layer | Anthropic API (Claude) | Structured feature extraction + natural-language explanations |
| Data handling | pandas, parquet for scored records | Lightweight, no dedicated DB needed at this scale |
| Fairness stats | Fairlearn / custom FDR-controlled testing | Rigorous subgroup significance testing, not just eyeballed rate differences |
| Dashboard | Streamlit | Fastest path from script to deployed, interactive app |
| Deployment | Streamlit Community Cloud / Render | Free, one-click deploy from GitHub |

## Datasets

- **Credit risk:** [LendingClub loan data](https://www.kaggle.com/datasets) (or equivalent public lending dataset)
- **Fraud detection:** [Kaggle credit card fraud dataset](https://www.kaggle.com/datasets)
- **Fairness audit:** demographic/protected attribute columns available in or synthetically joined to the above
